# Release Notes: BabeL-O v0.2.6

BabeL-O v0.2.6 is a major milestone that introduces **Native Single Executable Application (SEA)** distribution alongside a fully decoupled, headless Nexus backend server architecture.

---

## 简体中文 (Chinese)

### 📌 版本概述
BabeL-O v0.2.6 引入了基于 Node.js SEA (Single Executable Application) 的**单文件原生可执行二进制包分发**。此版本彻底贯彻了 “**Nexus 负责执行，CLI 负责交互**” 的设计理念，通过解耦底层引擎和终端界面，为用户提供零开销、亚秒级启动的通用泛化智能体体验。

### 🚀 核心特性与更新
1. **原生单文件二进制分发 (Node.js SEA)**
   - 支持一键编译生成 `dist/bbl`（Windows 系统为 `dist/bbl.exe`）单文件包。
   - **完全自包含**：目标运行环境无需安装 Node.js、`npm` 或 `node_modules` 依赖。
   - **内置资源整合**：项目内置的 `.md` 格式开发者技能文件直接作为原生 SEA Asset 注入二进制中，运行时从内存中动态提取。
   - **Homebrew 兼容性自动修复**：编译脚本能够自动检测本地 Node 是否缺少 Sentinel 标记（如 Homebrew 剥离符号的版本），并自动拉取缓存官方 Node runtime 模板完成安全注入。
   - **ESM require 兼容**：使用动态 `banner` 垫片技术，支持打包后的 ESM 包对 CommonJS 第三方库的加载。

2. **以 Nexus 为核心的通用泛化 Agent 架构**
   - 彻底移除了 React/Ink 渲染带来的资源消耗与启动延迟，使用原生 Readline 事件流。
   - 包含 `LocalCodingRuntime`（高效确定性本地调整）与 `LLMCodingRuntime`（动态多步推理与工具调用逻辑）双运行时。

3. **任务级 Git Worktree 编排与多智能体控制**
   - 多智能体以 **Planner ➔ Executor ➔ Critic** 流水线方式进行协同。
   - 使用文件系统排他锁（Mutex Locks）隔离并发任务，在独立的 Git 临时工作树中执行修改，合并前自动冲突检测，确保主分支安全。

4. **全面加强的安全隔离机制**
   - **路径安全隔离**：核心读写模块利用统一的路径解析和 symlink 拦截检测，杜绝任何路径穿越逃逸漏洞。
   - **Docker 隔离运行**：支持将 bash 命令执行放入受 CPU、内存、网络严格限制的 Docker 隔离沙箱。
   - **HMAC 指令校验**：本地非沙箱命令执行使用随机生成的 HMAC Token 进行校验，防御终端注入和 CWD 伪造攻击。

5. **结构化持久化审计与测试套件**
   - 所有的智能体事件、工具输入/输出、用量和授权审计日志全部写入 SQLite 本地数据库。
   - 全套 **350** 个关于上下文组装、分发、权限控制的自动化单元与集成测试均 100% 成功通过。

## English

### 📌 Overview
BabeL-O v0.2.6 introduces native **Single Executable Application (SEA)** distribution. Decoupled under the design philosophy of *"Nexus owns execution, CLI owns interaction"*, this release eliminates React/Ink CLI overhead, offering a lightning-fast, zero-overhead generalized agentic AI system.

### 🚀 Key Features & Enhancements
1. **Native Single Executable Application (Node.js SEA)**
   - Compile the entire application into a single standalone binary (`dist/bbl` / `dist/bbl.exe`).
   - **Zero dependency**: Runs on target machines without Node.js or `node_modules` pre-installed.
   - **Embedded Assets**: Built-in developer skill `.md` files are baked directly into the binary as native assets, loaded dynamically from memory.
   - **Automated Homebrew Workaround**: The build script automatically detects stripped Node.js runtimes (common on macOS Homebrew), pulls and caches official templates, and secures the binary sentinel injection.
   - **ESM require shim**: Integrates dynamic shimming to support CommonJS dependencies inside bundled ESM modules.

2. **Decoupled Generalized Agent Engine**
   - Fully eliminated heavy terminal wrappers, relying entirely on raw streams and readline events.
   - Powered by a dual-runtime layer: `LocalCodingRuntime` for deterministic local modifications and `LLMCodingRuntime` for dynamic agentic planning and tool execution loops.

3. **Multi-Agent Git Worktree Coordination**
   - Seamless agent coordination via a strict **Planner ➔ Executor ➔ Critic** pipeline.
   - Executes changes inside isolated temporary Git Worktrees secured by file mutex locks to prevent path overlap and merge conflicts.

4. **Security & Sandbox Isolation**
   - **Strict Path Safety**: Absolute path traversal prevention via symlink verification.
   - **Docker Sandbox**: Opt-in sandboxed shell execution constrained by CPU, memory, and networking limits.
   - **HMAC Command Nonces**: Local shell runs are signed with cryptographic HMAC tokens to protect against cwd-spoofing and script injections.

5. **Structured Audit Logs & Comprehensive Tests**
   - Persists all agent footprints, prompts, tokens, tool telemetry, and approval records to SQLite.
   - 100% success rate across all **350** core unit and integration tests.
