# Release Notes: BabeL-O v0.2.7

BabeL-O v0.2.7 is a minor release focusing on model matrix expansion, general AI agent positioning alignment, and build-system synchronization.

---

## 简体中文 (Chinese)

### 📌 版本概述
BabeL-O v0.2.7 在 v0.2.6 原生单文件可执行二进制打包的基础上，增加了新一代高性能大模型支持，优化了 TUI 终端自动补全的即时响应性，并在中英文档中全面推进了**通用泛化 AI 智能体（Generalized Agent）**的定位。

### 🚀 核心特性与更新
1. **模型矩阵新增 MiniMax-M3**
   - 官方集成并支持了 `minimax/MiniMax-M3`。
   - 该模型具备 200,000 Token 超长上下文窗口，支持原生流式输出（Streaming）、工具自动化调用（Tool Calling）以及结构化 JSON 输出，为长上下文复杂推理任务提供更优的选择。

2. **构建与全局命令同步优化**
   - 修复了本地 TypeScript 修改无法及时在全局软链 CLI 映射中生效的编译滞后问题。
   - 统一了本地编译生成物关系，确保 `/model` 下拉菜单与交互配置向导中模型列表的即时响应与一致性。

3. **通用泛化 AI 智能体定位对齐**
   - 全面更新了中英文 README 与发布说明口径，正式确立 BabeL-O 作为 **Nexus-first 通用泛化 AI 智能体** 的架构属性，强调其通过标准 MCP 协议和依赖解耦应对多场景任务的能力。

### 📦 安装与下载

#### 方法一：自动脚本一键安装（推荐，最快）
在 macOS 或 Linux 终端中运行以下命令，即可自动检测系统架构、下载对应的原生二进制文件并配置好执行路径：
```bash
curl -fsSL https://raw.githubusercontent.com/SuTang-vain/BabeL-O/main/scripts/install.sh | bash
```

#### 方法二：手动下载原生二进制包
如果无法使用自动脚本，可以直接点击以下链接手动下载对应系统的单文件二进制：
- **macOS (Apple Silicon)**: [bbl-darwin-arm64](https://github.com/SuTang-vain/BabeL-O/releases/download/v0.2.7/bbl-darwin-arm64)
- **macOS (Intel Core)**: [bbl-darwin-x64](https://github.com/SuTang-vain/BabeL-O/releases/download/v0.2.7/bbl-darwin-x64)
- **Linux (x64)**: [bbl-linux-x64](https://github.com/SuTang-vain/BabeL-O/releases/download/v0.2.7/bbl-linux-x64)
- **Windows (x64)**: [bbl-windows-x64.exe](https://github.com/SuTang-vain/BabeL-O/releases/download/v0.2.7/bbl-windows-x64.exe)

*注：手动下载后，macOS/Linux 用户请执行 `chmod +x <filename>` 赋予可执行权限，并将其移动到系统的 `$PATH` 变量路径下（如 `/usr/local/bin/bbl`）。*

## English

### 📌 Overview
BabeL-O v0.2.7 introduces support for MiniMax's next-generation M3 model, aligns TUI autocomplete syncing, and completes the terminology updates defining BabeL-O as a generalized AI agent.

### 🚀 Key Features & Enhancements
1. **Added MiniMax-M3 Support**
   - Native registry and routing for `minimax/MiniMax-M3`.
   - Offers a 200,000 Token context window, tool calling stability, JSON output structuring, and fast streaming capability.

2. **Synchronized Build System for Global CLI Links**
   - Corrected build sync logic where updates to `registry.ts` weren't instantly reflected in the globally linked TUI.
   - Synchronized the active model list in the `/model` dropdown and terminal autocomplete menu.

3. **Terminology Alignment for Generalized AI Agents**
   - Completed updates to all project files and README docs to position BabeL-O as a **Nexus-first generalized AI agentic system**, highlighting its versatility outside of strictly coding environments via MCP servers.
