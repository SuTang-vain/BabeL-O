# Release Notes: BabeL-O v0.2.8

BabeL-O v0.2.8 is a runtime hardening release focused on cache-aware context management, richer context diagnostics, TUI polish, and CI/build quality gates.

---

## 简体中文 (Chinese)

### 版本概述

v0.2.8 将上下文管理从固定阈值推进到更缓存感知的策略：在保护 context-window 安全边界的同时，减少不必要的 compact，降低首字延迟，并更充分地利用 Anthropic / MiniMax 等大上下文模型的 prompt / prefetch cache 与长窗口能力。

### 核心特性与更新

1. **Cache-aware Compact / 长上下文利用**
   - 新增 cache-aware compact policy，用 cache read/create tokens、cacheable system prompt ratio、reserved output、provider safety buffer 和 recent provider context error 共同决定 effective context ceiling。
   - 大上下文模型可突破旧的 120k legacy ceiling，在保留输出空间和安全 buffer 的前提下使用更高有效上下文。
   - `/context` 诊断新增 cache economics：cache read ratio、cache-preserving mode、long-context utilization mode、effective/legacy ceiling、compact reason。

2. **Runtime diagnostics 与 provider recovery 增强**
   - 新增统一 runtime diagnostics envelope，让 context/provider/fallback 诊断更结构化。
   - provider fallback、context blocking、max output recovery 与 malformed/empty provider response 的可观测性继续增强。
   - Compact summary latency、cache usage、microcompact savings 与 runtime metrics 可被 benchmark/diagnostics 使用。

3. **TUI / CLI 交互增强**
   - `/context` 进入可滚动 context view，支持 summary / expanded diagnostics 视图。
   - 新增 tool audit formatter，并在 `/status` 中展示工具可用性、风险等级与来源。
   - 改进 slash/tool palette、Shift+Enter 多行输入、paste/input 状态、agent running/status 渲染与 AgentLoop TUI smoke。

4. **Session Memory / Compact / Tool Result Budget**
   - Session Memory Lite 增加更新状态、诊断、自然停顿策略与成本控制信息。
   - Compact post-restore、microcompact、tool result budget 与 Read 工具 recoverable error 覆盖继续增强。
   - Runtime pipeline / tool loop 进一步拆分，降低 `LLMCodingRuntime` 主循环复杂度。

5. **工程化质量门禁**
   - 新增 CI workflow。
   - 新增 `format:check`、`lint`、`coverage`、`build:smoke`、`deps:audit`、`test:performance` 脚本。
   - 新增 dependency boundary audit、coverage report、production build smoke 与更完整 benchmark 输出。

## English

### Overview

v0.2.8 advances BabeL-O's context stack from fixed thresholds toward cache-aware compact decisions. It preserves safety around context-window limits while reducing unnecessary compaction, lowering first-token latency risk, and improving long-context utilization for Anthropic / MiniMax-class models.

### Key Features & Enhancements

1. **Cache-aware Compact & Long-context Utilization**
   - Added cache-aware compact policy based on cache read/create tokens, cacheable system prompt ratio, reserved output, provider safety buffer, and recent provider context errors.
   - Large-context models can use an effective ceiling above the old 120k legacy cap while reserving output and safety buffer.
   - `/context` now surfaces cache economics including cache-preserving mode, long-context mode, effective/legacy ceiling, and compact policy reason.

2. **Runtime Diagnostics & Provider Recovery**
   - Added a shared runtime diagnostics envelope for context/provider/fallback diagnostics.
   - Improved observability for provider fallback, context blocking, max output recovery, malformed provider responses, and empty responses.
   - Compact summary latency, cache usage, microcompact savings, and runtime metrics are now benchmark/diagnostic friendly.

3. **TUI / CLI Improvements**
   - `/context` opens an interactive context view with summary and expanded diagnostics.
   - `/status` now includes formatted tool audit details.
   - Improved slash/tool palettes, Shift+Enter multiline input, paste/input state handling, agent status rendering, and AgentLoop TUI smoke coverage.

4. **Session Memory / Compact / Tool Result Budget**
   - Session Memory Lite now exposes richer status, diagnostic, natural-pause, and cost-control metadata.
   - Compact post-restore, microcompact, tool result budget, and recoverable Read errors gained additional coverage.
   - Runtime pipeline and tool loop helpers were further split out from `LLMCodingRuntime`.

5. **Quality Gates**
   - Added CI workflow.
   - Added `format:check`, `lint`, `coverage`, `build:smoke`, `deps:audit`, and `test:performance`.
   - Added dependency boundary audit, coverage report, production build smoke, and broader benchmark output.
