# Release Notes: BabeL-O v0.2.9

BabeL-O v0.2.9 is an orchestration and observability release focused on persistent Agent jobs, richer runtime metrics, provider compatibility hardening, benchmark history, and optional runner comparison diagnostics.

---

## 简体中文 (Chinese)

### 版本概述

v0.2.9 继续推进 BabeL-O 的 Nexus-first 多 Agent 基础设施：AgentScheduler 从内存态能力迈向可持久、可治理、可观测的 Agent job 系统，同时补强 provider 协议兼容、runtime metrics、benchmark history 和并发测试烟测，为后续更复杂的 Explore / Review / Test / Implement Agent 打地基。

### 核心特性与更新

1. **Agent Job 持久化与治理**
   - Agent job 类型抽到共享层，支持持久化、恢复、过滤、transcript 查询和 job lifecycle event。
   - AgentScheduler 增加并发上限、深度限制、运行超时、timeoutAt 与 governance 元数据。
   - Agent API / CLI 能更稳定地 spawn、list、wait、cancel，并在父会话中保留 child session / transcript 引用。

2. **Agent / Runtime 可观测性**
   - `/v1/runtime/metrics` 增加 provider invocation、AgentLoop 和 AgentJob 聚合指标。
   - AgentLoop role step metrics 增加 input/output tokens、duration、failure kind / error code 等诊断。
   - TUI/CLI 事件渲染继续增强 Agent job、status 和 context 展示，减少长任务中的状态黑盒。

3. **Provider 协议兼容增强**
   - 增强 Anthropic-compatible、MiniMax、DeepSeek / OpenAI-compatible adapter 的 tool call / reasoning / usage 解析。
   - 补充 provider registry capability 诊断，减少模型协议细节泄漏到 runtime 主循环。
   - 强化 provider recovery 测试，覆盖 malformed tool calls、text-encoded tool calls、empty response、max output 和 protocol mismatch。

4. **Benchmark History 与 Retry Policy Benchmark**
   - `npm run benchmark` 新增本地 benchmark history，默认写入 `.babel-o/benchmarks/`，并加入 `.gitignore`。
   - 新增 retry policy benchmark，统计 rate limit、provider unavailable、empty response、schema mismatch、tool protocol error 的 retry/repair 成本。
   - benchmark 输出增加历史 delta、关键指标摘要和 runtime metrics 快照。

5. **Runner Comparison / Go Runner 诊断**
   - 新增 TS local runner 与可选 Go RemoteToolRunner 对比 benchmark。
   - 覆盖 Read、Grep、Glob、Bash stdout、output truncation、timeout、cancel latency 和 workspace escape。
   - Go runner benchmark 默认跳过，需显式设置 `BABEL_O_RUN_GO_RUNNER_SMOKE=1`。

6. **测试与并发治理**
   - 新增 `npm run test:concurrency` 并发 smoke，按独立临时 config 运行可并发测试文件。
   - 默认测试纳入 benchmark history、retry policy、runner comparison、Agent job persistence 和 provider compatibility 回归。
   - 继续保持默认无 Go 依赖；Go runner smoke 与 benchmark 仍需显式 opt-in。

## English

### Overview

v0.2.9 advances BabeL-O's Nexus-first orchestration foundation. AgentScheduler now moves from in-memory behavior toward persistent, governable, observable Agent jobs, while provider compatibility, runtime metrics, benchmark history, runner comparison diagnostics, and concurrency smoke coverage are tightened for future Explore / Review / Test / Implement agents.

### Key Features & Enhancements

1. **Persistent and Governed Agent Jobs**
   - Moved Agent job contracts into shared types with storage persistence, restore, filtering, transcript lookup, and lifecycle events.
   - Added scheduler governance metadata: max concurrency, depth limits, runtime limits, timeoutAt, and active agent counts.
   - Agent API / CLI paths can more reliably spawn, list, wait, and cancel jobs while preserving child session and transcript references.

2. **Agent / Runtime Observability**
   - `/v1/runtime/metrics` now includes provider invocation, AgentLoop, and AgentJob aggregate metrics.
   - AgentLoop role step metrics include input/output tokens, duration, failure kind, and error code diagnostics.
   - CLI/TUI event rendering gained clearer Agent job, status, and context visibility for long-running work.

3. **Provider Protocol Hardening**
   - Improved Anthropic-compatible, MiniMax, DeepSeek, and OpenAI-compatible parsing for tool calls, reasoning, and usage events.
   - Expanded provider registry capability diagnostics to keep protocol details out of the runtime loop.
   - Added regression coverage for malformed tool calls, text-encoded tool calls, empty responses, max output recovery, and protocol mismatch.

4. **Benchmark History and Retry Policy Benchmark**
   - `npm run benchmark` now writes local benchmark history under `.babel-o/benchmarks/` by default.
   - Added retry policy benchmark coverage for rate limits, provider unavailable errors, empty responses, schema mismatch, and tool protocol errors.
   - Benchmark output now includes historical deltas, key metric summaries, and runtime metrics snapshots.

5. **Runner Comparison / Go Runner Diagnostics**
   - Added TS local runner versus optional Go RemoteToolRunner comparison benchmark.
   - Covers Read, Grep, Glob, Bash stdout, output truncation, timeout correctness, cancel latency, and workspace escape.
   - Go runner benchmark remains opt-in via `BABEL_O_RUN_GO_RUNNER_SMOKE=1`.

6. **Testing and Concurrency Governance**
   - Added `npm run test:concurrency` for isolated concurrent smoke runs.
   - Default tests now cover benchmark history, retry policy, runner comparison, Agent job persistence, and provider compatibility.
   - Default test/build flows still do not require Go; Go runner smoke and benchmarks remain explicit opt-in paths.
