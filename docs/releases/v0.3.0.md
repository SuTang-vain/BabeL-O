# Release Notes: BabeL-O v0.3.0

BabeL-O v0.3.0 is a polish and usability release focused on brand assets, README install accuracy, TUI context input, file/symbol references, safer file discovery, and the `docs/nexus` documentation system.

---

## English

### Overview

v0.3.0 makes BabeL-O easier to install, easier to recognize, and easier to drive in long coding sessions. The README now shows the BabeL-O product logo and KezhongKe IP brand logo, with technical support credited to **KezhongKe (壳中客)**. The CLI also gains more practical context-entry affordances and safer discovery behavior.

### Key Updates

1. **Brand and README Refresh**
   - Added the BabeL-O product logo to `docs/assets/babel-o-logo.png`.
   - Added the KezhongKe IP brand logo to `docs/assets/kezhongke_logo_3d.png`.
   - Updated English and Chinese READMEs to show both logos and credit technical support to KezhongKe (壳中客).
   - Updated README release links to this v0.3.0 release note.

2. **TUI Context Input Improvements**
   - Added file attachment references for prompt-time context injection.
   - Added image reference metadata handling without changing provider message schemas.
   - Added lightweight LSP-style symbol and diagnostic mentions for source navigation.
   - Added opt-in vim input mode through `BABEL_O_VIM_MODE=1`.

3. **Safer Tool Discovery**
   - Added optional bundled ripgrep support through `@vscode/ripgrep`, used before system `rg`.
   - Improved `Grep` fallback behavior and no-result diagnostics.
   - Hardened `scripts/install.sh` so failed release asset downloads stop the installer instead of installing an error response.
   - Added Bash-as-file-discovery guidance so broad `ls` / `find` / `tree` / recursive grep style commands steer models toward `ListDir`, `Glob`, `Grep`, and `Read`.
   - Kept ordinary Bash timeout / SIGTERM failures recoverable instead of turning them into fatal session errors.

4. **Docs/Nexus Structure**
   - Reorganized `docs/nexus` into `active/`, `reference/`, and `archive/`.
   - Added README files for each docs layer.
   - Updated `.gitignore` so `docs/nexus/**/*.md` and `docs/assets/*.png` are tracked intentionally while other local planning files remain ignored.

### Installation

The following installation paths are backed by files and workflows in this repository.

#### Option A: Install the latest release binary

The installer script detects macOS or Linux architecture, reads the latest GitHub release tag, downloads the matching `bbl-*` asset, and installs it as `bbl`.

```bash
curl -fsSL https://raw.githubusercontent.com/SuTang-vain/BabeL-O/main/scripts/install.sh | bash
bbl chat
```

Notes:

- Supported by the installer: macOS arm64, macOS x64, Linux x64.
- Windows users should use the manual release asset or build from source.
- Release assets are produced by `.github/workflows/release.yml` when the `v0.3.0` tag is pushed.

#### Option B: Manually download a release binary

After the GitHub release workflow completes, these assets are expected for v0.3.0:

- [macOS Apple Silicon: bbl-darwin-arm64](https://github.com/SuTang-vain/BabeL-O/releases/download/v0.3.0/bbl-darwin-arm64)
- [macOS Intel: bbl-darwin-x64](https://github.com/SuTang-vain/BabeL-O/releases/download/v0.3.0/bbl-darwin-x64)
- [Linux x64: bbl-linux-x64](https://github.com/SuTang-vain/BabeL-O/releases/download/v0.3.0/bbl-linux-x64)
- [Windows x64: bbl-windows-x64.exe](https://github.com/SuTang-vain/BabeL-O/releases/download/v0.3.0/bbl-windows-x64.exe)

On macOS/Linux:

```bash
chmod +x ./bbl-*
mv ./bbl-* /usr/local/bin/bbl
bbl chat
```

#### Option C: Build from source

This path is immediately verifiable from the repository and does not depend on release assets.

```bash
git clone https://github.com/SuTang-vain/BabeL-O.git
cd BabeL-O
npm ci
npm run build
npm run cli -- --help
npm run cli -- run "hello"
```

For local CLI development:

```bash
npm link
bbl chat
```

To build a standalone native binary locally:

```bash
npm run build:binary
./dist/bbl chat
```

## 简体中文

### 版本概述

v0.3.0 聚焦品牌展示、README 安装口径、TUI 上下文输入、文件/符号引用、安全文件发现，以及 `docs/nexus` 文档体系整理。README 现在展示 BabeL-O 产品 logo 与 KezhongKe IP 品牌 logo，并明确由 **KezhongKe（壳中客）** 提供技术支持。

### 核心更新

1. **品牌与 README 更新**
   - 新增 BabeL-O 产品 logo：`docs/assets/babel-o-logo.png`。
   - 新增 KezhongKe IP 品牌 logo：`docs/assets/kezhongke_logo_3d.png`。
   - 英文和中文 README 顶部展示两个 logo，并补充 KezhongKe（壳中客）技术支持说明。
   - README 中的版本发布说明链接更新到 v0.3.0。

2. **TUI 上下文输入增强**
   - 支持 prompt 提交前的文件附件引用。
   - 支持图片引用 metadata，不改变 provider message schema。
   - 支持轻量 `@symbol:` / `@diagnostic:` 上下文提及。
   - 支持 `BABEL_O_VIM_MODE=1` 开启 opt-in vim 输入模式。

3. **更安全的工具发现**
   - 新增 optional bundled ripgrep：优先使用 `@vscode/ripgrep`，再退回系统 `rg`，最后使用 JS fallback。
   - 增强 `Grep` fallback 和 no-result diagnostics。
   - 加固 `scripts/install.sh`，release asset 下载失败时直接停止安装，避免把错误响应写成可执行文件。
   - 对 broad `ls` / `find` / `tree` / recursive grep 类 Bash 文件发现命令增加 guidance，引导模型优先使用 `ListDir`、`Glob`、`Grep`、`Read`。
   - Bash timeout / SIGTERM 保持可恢复工具失败，不升级为会话 fatal error。

4. **docs/nexus 文档结构整理**
   - 将 `docs/nexus` 分为 `active/`、`reference/`、`archive/`。
   - 为各层新增 README。
