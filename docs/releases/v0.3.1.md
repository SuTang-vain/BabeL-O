# Release Notes: BabeL-O v0.3.1

BabeL-O v0.3.1 is a patch release focused on making the published standalone binaries installable and executable.

---

## English

### Overview

v0.3.1 fixes the release binary build path that caused the published `bbl` executable to fail at startup with `SyntaxError: Identifier 'createRequire' has already been declared`. It also keeps the hardened release installer behavior so failed or incomplete downloads cannot be installed as `bbl`.

### Fixes

1. **Standalone binary startup fix**
   - Fixed the SEA binary bundle banner in `scripts/build-binary.js`.
   - The esbuild banner now imports `createRequire` as `babelOCreateRequire`, avoiding a duplicate top-level ESM declaration when bundled code also imports `createRequire`.
   - Added a focused regression test to keep the banner aliased.

2. **Installer hardening retained**
   - `scripts/install.sh` supports `BBL_VERSION=v0.3.1` for explicit version installs.
   - Downloads are written to a temporary file and only replace `bbl` after validation.
   - The installer retries downloads, checks `Content-Length` when GitHub provides it, and validates executable magic before installation.
   - HTTP error bodies such as `Not Found` are rejected instead of being installed.

### Installation

Install the latest release binary on macOS or Linux:

```bash
curl -fsSL https://raw.githubusercontent.com/SuTang-vain/BabeL-O/main/scripts/install.sh | bash
bbl chat
```

Install this release explicitly:

```bash
curl -fsSL https://raw.githubusercontent.com/SuTang-vain/BabeL-O/main/scripts/install.sh | BBL_VERSION=v0.3.1 bash
bbl chat
```

## 简体中文

### 版本概述

v0.3.1 是一个补丁版本，重点修复发布二进制 `bbl` 启动时报 `SyntaxError: Identifier 'createRequire' has already been declared` 的问题，并保留安装脚本的下载校验加固，避免错误响应或不完整下载被安装成可执行文件。

### 修复内容

1. **单文件二进制启动修复**
   - 修复 `scripts/build-binary.js` 中的 SEA binary bundle banner。
   - esbuild banner 现在把 `createRequire` 导入为 `babelOCreateRequire`，避免打包后的代码中已有 `createRequire` 导入时出现顶层 ESM 重复声明。
   - 增加 focused 回归测试，防止后续重新引入同名 banner。

2. **保留安装脚本加固**
   - `scripts/install.sh` 支持 `BBL_VERSION=v0.3.1` 指定版本安装。
   - 下载先写入临时文件，验证通过后才替换 `bbl`。
   - 安装脚本会重试下载、在 GitHub 返回 `Content-Length` 时校验大小，并在安装前校验可执行文件 magic。
   - `Not Found` 等 HTTP 错误响应会被拒绝安装。
