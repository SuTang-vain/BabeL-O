# Release Notes: BabeL-O v0.3.2

BabeL-O v0.3.2 is a feature release that adds first-class **SessionChannel collaboration** to the TUI client and CLI, makes the interactive chat safer to launch under non-TTY conditions, and hardens the tool surface (ListDir / Grep / Read) with explicit workspace drift detection. The release is still served by the hardened `scripts/install.sh` and ships the same standalone-binary assets as v0.3.1.

---

## English

### Overview

v0.3.2 promotes the in-development SessionChannel work to a user-visible capability. The chat loop can now preview, queue, and confirm side-channel messages, visualize a session tree, inspect a debug-only channel graph, and open a unified **Collaborate** hub that lists unread inbox messages, active channels, and child / related agent jobs. v0.3.2 also tightens the v0.3.1 release fix: `bbl chat` now fails clearly when stdin/stdout are not TTYs, and the standalone binary keeps an idle event loop so the chat prompt does not disappear.

### Highlights

1. **SessionChannel inbox in the chat loop**
   - `/inbox` opens an overlay listing unread `SessionMessage` items for the current session.
   - `/inbox reply channel=<id> message=<id> [type=answer] [priority=normal] -- <message>` previews a reply, then `/inbox reply confirm` sends it.
   - Incoming `permission_request`, `permission_response`, `review_request`, `handoff`, and `blocked` events render as inline footer cards without blocking the prompt.
   - `bbl sessions inbox <sessionId>` exposes the same inbox outside the TUI for CI / headless consumers.
   - `bbl sessions ack <sessionId> <messageId>` marks an inbox message as acknowledged.

2. **Typed side-channel send with explicit confirm step**
   - `/channel send channel=<id> [to=<sessionId>|broadcast=true] [type=question|finding|request_review|request_validation|hypothesis|decision|blocked|memory_candidate|handoff] [priority=low|normal|high] -- <message>` first renders a preview, then `/channel send confirm` actually sends it.
   - `/channel send cancel` discards the pending draft.
   - Preview, send, and cancel flows are intentionally separated so an agent can never dispatch a side-channel message from raw pasted text without a human-confirmed `confirm` step.

3. **Debug-only channel topology view**
   - `/channels graph` renders a read-only `SessionChannel` topology rooted at the current session.
   - The view is clearly labeled as a debug topology (not a transcript, not direct user instructions) and never executes any side effect.

4. **Collaborate hub overlay**
   - `/collaborate` opens a single hub with three entry points: **Inbox**, **Channels**, and **Agents** (parent / child / related jobs).
   - Each entry shows counts and a short hint, and the hub routes to the matching overlay or to the channel-send composer.

5. **Activity overlay**
   - `/activity` shows the most recent `SessionChannel` activity for the current session, including acks, decisions, and handoffs.
   - Activity is context, not instructions: cards carry a "verify before acting" note, and the hub prompts the user to open the inbox before acting on a finding.

6. **Hardened chat launch and idle loop**
   - `bbl chat` (and `bbl chat dev`) refuses to start when either stdin or stdout is not a TTY and prints a clear error pointing to `bbl run "<prompt>"` for non-interactive prompts.
   - The chat loop calls `input.resume()` and keeps a no-op interval so the standalone-binary (SEA) event loop does not exit during idle moments after the welcome card.

7. **Tool boundary tightening**
   - `pathDrift` checks now apply uniformly to `ListDir`, `Grep`, and `Read`. Out-of-workspace symlink escapes are detected before any FS read.
   - Grep and Read return a structured `pathSafety.drift` block so the agent can surface the underlying real path without bypassing the safety check.
   - New MCP tool adapters are introduced one capability at a time, keeping each tool's responsibility orthogonal (no broad "do anything" wrappers).

8. **Installer and CLI scaffolding**
   - `scripts/install.sh` still validates `Content-Length` and executable magic, but the fallback hint is bumped to `BBL_VERSION=v0.3.2`.
   - `npm run cli -- chat dev` from a source checkout still shows the `❖ BABEL-O  dev` TUI title so contributors can distinguish local builds from installed binaries.

### Installation

Install the latest release binary on macOS or Linux:

```bash
curl -fsSL https://raw.githubusercontent.com/SuTang-vain/BabeL-O/main/scripts/install.sh | bash
bbl chat
```

Install this release explicitly:

```bash
curl -fsSL https://raw.githubusercontent.com/SuTang-vain/BabeL-O/main/scripts/install.sh | BBL_VERSION=v0.3.2 bash
bbl chat
```

## 简体中文

### 版本概述

v0.3.2 是一个功能版本，将开发中的 SessionChannel 协作能力正式暴露给终端用户：在 `bbl chat` 中可预览、确认并发送有类型的 side-channel 消息，查看 session 树与 channel 拓扑，并通过统一的 **Collaborate** 入口集中查看 Inbox / Channels / 子智能体。v0.3.2 同样保留了 v0.3.1 的非交互终端守护：在非 TTY 场景下 `bbl chat` 会清晰报错并指向 `bbl run`，同时 SEA 单文件二进制在闲置时不会因为事件循环为空而退出。

### 主要更新

1. **TUI 内置 SessionChannel Inbox**
   - `/inbox` 打开当前 session 的未读 `SessionMessage` 收件箱。
   - `/inbox reply channel=<id> message=<id> [type=answer] [priority=normal] -- <message>` 先生成预览，再通过 `/inbox reply confirm` 真正发送。
   - `permission_request`、`permission_response`、`review_request`、`handoff`、`blocked` 等事件以内联卡片形式直接出现在输入框下方，不阻塞 prompt。
   - `bbl sessions inbox <sessionId>` 在 TUI 之外以 JSON / 文本形式暴露同一份收件箱，方便 CI / 无头调用。
   - `bbl sessions ack <sessionId> <messageId>` 用于显式确认消息。

2. **类型化 side-channel 发送 + 显式确认**
   - `/channel send channel=<id> [to=<sessionId>|broadcast=true] [type=question|finding|request_review|request_validation|hypothesis|decision|blocked|memory_candidate|handoff] [priority=low|normal|high] -- <message>` 先渲染预览，必须再执行 `/channel send confirm` 才真正发送。
   - `/channel send cancel` 丢弃待发草稿。
   - 预览 / 发送 / 取消三步分离，保证任何 side-channel 消息的发出都来自 human-confirmed 的 `confirm`，杜绝粘贴文本被当成 prompt 误触发发送。

3. **调试用 Channel 拓扑图**
   - `/channels graph` 以当前 session 为根渲染只读 `SessionChannel` 拓扑。
   - 该视图顶部明确标注 "debug-only topology view"，不是 transcript、不是用户指令，也不会执行任何副作用。

4. **Collaborate 协作中心**
   - `/collaborate` 打开统一入口，包含 Inbox / Channels / Agents 三个子项。
   - 每个子项都展示数量与简短说明，并跳转到对应的 overlay 或 channel-send 编辑器。

5. **Activity 活动面板**
   - `/activity` 展示当前 session 最近的 `SessionChannel` 活动，包括 ack、decision、handoff 等。
   - 活动面板中的卡片始终携带 "verify before acting" 提示，并引导用户先打开 Inbox 再据此采取行动。

6. **更稳的 chat 启动与闲置循环**
   - 当 stdin / stdout 任意一侧不是 TTY 时，`bbl chat` (含 `bbl chat dev`) 立即报错并提示改用 `bbl run "<prompt>"`。
   - 启动时调用 `input.resume()` 并保留一个空操作定时器，确保 SEA 单文件二进制在闲置状态下不会因为事件循环为空而提前退出。

7. **工具边界收紧**
   - `pathDrift` 检查统一覆盖 `ListDir` / `Grep` / `Read` 三个内置工具，越界 symlink 会在任何 FS 读取前被拦截。
   - Grep / Read 工具的结果中带结构化 `pathSafety.drift` 字段，便于上层在不绕过安全检查的前提下提示真实物理路径。
   - 新增的 MCP 工具适配器按能力一项一项引入，单个工具职责正交，不再有"什么都做"的宽泛包装器。

8. **安装脚本与 CLI 脚手架**
   - `scripts/install.sh` 仍会校验 `Content-Length` 与可执行 magic，错误提示中的版本号同步更新为 `BBL_VERSION=v0.3.2`。
   - 在源码目录下 `npm run cli -- chat dev` 仍然显示 `❖ BABEL-O  dev` 标题，方便贡献者区分本地构建与已安装的二进制。
